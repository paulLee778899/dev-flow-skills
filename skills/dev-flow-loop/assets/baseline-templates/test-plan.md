## 3.6 资源约束验证（嵌入式，如适用）

> 非嵌入式项目可删除本节。

**目标**：验证实现后各模块和整系统的 RAM、Stack、Code/Flash 占用不超过概要设计 §6.3 规定的预算上限。

| 验证方向 | 验证方法 | 通过标准 | 工具 |
| --- | --- | --- | --- |
| RAM 占用 | 编译后分析 Linker Map（.map 文件）的 `.data` + `.bss` 段 | 总 RAM ≤ {{X}} KB | `arm-none-eabi-size` + 手工核算 |
| Code/Flash 占用 | 分析 Linker Map 的 `.text` + `.rodata` 段 | 总 Code ≤ {{Z}} KB | `arm-none-eabi-size` / Keil Build Output |
| 栈深度 | 静态分析每个任务的最深调用链，累加局部变量 | 每任务栈 ≤ 设计预算；总 Stack ≤ {{Y}} KB | StackAnalyzer / 手动计算 + 代码审查 |
| 动态内存使用 | 代码扫描，确认运行时无 `malloc`/`free` | 无运行时动态分配 | grep 脚本 + 静态分析（PC-lint / cppcheck） |
| 数据类型合规 | 静态分析检查接口字段类型 | 无 `int`/`long` 作为接口字段；无隐式类型转换警告 | `-Wall -Wextra -Werror` + MISRA 检查 |
| 静态断言验证 | 编译时 `_Static_assert` 检查结构体尺寸 | 编译通过，无断言失败 | 编译器内置 |
